# 修复报告：A/B 类计划混合显示问题

## 问题描述

用户在"智能模拟"功能中选择"A 类（成绩体系）"或"B 类（成绩体系）"时，系统没有正确区分"A 类计划"和"B 类计划"的分数线，导致两种计划类型的学校混合显示。

## 根本原因

1. **数据问题**：原始 Excel 数据中的"计划属性"列为空值（/），没有标识 A 类计划或 B 类计划
2. **查询逻辑问题**：后端代码尝试用 `substr(plan_type, 1, 1) = ?` 来匹配 'A' 或 'B'，但数据库中的 `plan_type` 是"普通高中"、"指标生"等，不是"A 类计划"或"B 类计划"

## 解决方案

### 1. 数据导入逻辑修改

修改 `import_excel.py`，为所有普通高中的记录创建两份：
- 一份标记为"A 类计划"
- 一份标记为"B 类计划"

这样 A 类和 B 类考生都能看到所有普通高中学校，但系统会正确区分计划类型。

### 2. 前端显示逻辑修复

修复 `app/templates/simulate.html` 中的逻辑：
- 根据当前模式（A/B）获取对应的结果数据
- 正确处理 C 等级情况
- 确保 A 类考生可以看到中职学校推荐

### 3. 数据验证

导入后的数据：
- 总记录数：3909 条
- A 类计划：140 条
- B 类计划：140 条
- 中职学校：1154 条
- 其他计划类型（指标生、艺术生等）：2575 条

## 修改的文件

1. `import_excel.py` - 新增数据导入脚本
2. `app/templates/simulate.html` - 修复前端显示逻辑
3. `.gitignore` - 添加数据库文件忽略规则
4. `data/分数线数据库.xlsx` - 更新为最新数据
